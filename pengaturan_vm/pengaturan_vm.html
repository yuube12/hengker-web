1. NAT (Network Address Translation)

Penjelasan:

  • VM bisa internetan.
  • Terisolasi dari VM lain dan dari jaringan lokal.
  • Tidak bisa di-scan dari luar (kecuali ada port forwarding).

Digunakan saat:

  • Install tools, update, browsing.
  • Ingin aman dari luar, dan tidak perlu komunikasi antar VM.

Kekurangan: Tidak sepenuhnya anonim. Trafik tetap lewat IP asli host dan ISP tetap bisa melihat.

2. NAT Network

Penjelasan:

  • Mirip NAT, tapi antar VM bisa saling terhubung.
  • Masih bisa internetan.
  • Harus setup NAT Network di VirtualBox Preferences.

Digunakan saat:

  • Simulasi jaringan lokal antar beberapa VM (attacker & korban).
  • CTF, lab malware multi-host, MITM antar VM.

Kekurangan: Masih tidak sepenuhnya anonim ke ISP, kecuali pakai VPN/Tor di dalam VM.

3. Bridged Adapter

Penjelasan:

  • VM terhubung langsung ke jaringan fisik seperti komputer biasa.
  • Dapat IP dari router, bisa diakses dari perangkat lain.

Digunakan saat:

  • Butuh VM terlihat di jaringan nyata (misal: diserang dari perangkat lain).
  • Pentest target nyata di jaringan lokal.

Kekurangan: Paling tidak anonim. VM muncul sebagai perangkat nyata di LAN, mudah dilacak via MAC & IP router.

4. Not Attached

Penjelasan:

  • VM tidak punya koneksi jaringan sama sekali.
  • Seperti komputer tanpa LAN/Wi-Fi.

Digunakan saat:

  • Latihan pentesting offline.
  • Isolasi total (misalnya analisis malware).

Kelebihan: Paling anonim, tapi tidak bisa browsing atau akses luar.
